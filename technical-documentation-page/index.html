Introduction

TypeScript is a strongly typed programming language that builds on JavaScript, giving you better tooling at any scale.

It adds optional static types to JavaScript, which can help prevent many common errors during development.

Here's why you might want to use TypeScript:

  • Static typing
  • Object-oriented features
  • IDE support
  • ECMAScript compatibility
  • Cross-platform and cross-browser compatibility
// TypeScript code const message: string = "Hello, TypeScript!";
Basic Types

TypeScript supports several basic types that you can use to define variables.

The most common types include:

let isDone: boolean = false; let decimal: number = 6; let color: string = "blue"; let list: number[] = [1, 2, 3];

You can also use type inference to let TypeScript determine the type automatically.

  • Boolean - true/false values
  • Number - floating point values
  • String - textual data
Interfaces

Interfaces are a powerful way to define contracts in your code and to define custom types.

They can describe object shapes, function types, and even classes.

interface User { name: string; id: number; }

You can extend interfaces to create more specific types:

interface Admin extends User { role: string; }
Functions

TypeScript functions can be typed with parameter types and return types.

Function types can be specified using interfaces or type aliases.

function add(x: number, y: number): number { return x + y; }

Arrow functions are also supported with type annotations:

  • Parameter types
  • Return types
  • Optional parameters
  • Default parameters
Classes

TypeScript offers full support for classes introduced in ES2015.

Classes in TypeScript can include:

  • Constructors
  • Properties
  • Methods
class Animal { name: string; constructor(name: string) { this.name = name; } }

TypeScript adds type annotations and other syntax to allow you to express relationships between classes and other types.

Generics

Generics allow you to write flexible, reusable code that works with multiple types.

They provide a way to make components work with any data type and not restrict to one type.

function identity(arg: T): T { return arg; }

Common use cases for generics include:

  • Arrays
  • Promises
  • Custom data structures
